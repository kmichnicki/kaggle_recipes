Let's state the problem. Given a list of ingredients, we need to pick one cuisine out of 20. So the algorithm gets ingredients and spits out a prediction. 
​
One simple idea is assign a conditional probability for a particular ingredient showing up given the cuisine. We then use bayes rule to update our belief about the cuisine given a prior distribution that matches the frequency of each cuisine in the training data. Then the product that is the highest wins. One problem with this approach is that there may be a new ingredient that we've never seen and we don't want to say that the probability is zero. We would just omit it. What if an ingredient shows up for a cuisine and all of the ingredients look brazilian except there is one ingredient that has never shown up in brazilian cuisine, since our brazilian sample is small this is likely, and yet it shows up in a different cuisine which otherwise is a poor choice. We wouldn't want to throw brazilian out as a candidate just because of one ingredient. Or would we? At the very least, we would want some way of not throwing away possibilities because of a single discrepency, especially if the competing candidates are bad. 
​
A problem with this simple bayesian update is that it ignores pairs of data. Maybe if flour is in the list of ingredients, sugar will be there too and if sugar is not there, it is much more information than if it were not there and we didn't know if flour was there. However considering 3, 4 and higher body effects would be computationally prohibative. 
​

def construct_pairs(ingredients):
    ingredients2=[i for i in ingredients]
    output=[]
    for i,v1 in enumerate(ingredients):
        for j,v2 in enumerate(ingredients2):
            if i<j:
                output.append(v1+str(",")+v2)
    return output

So it looks like a lot of recipes have similar preferences for word order. For instance, salt seems to come before onions pretty consistently. However, if we were to see onions come before salt, then what? Which cuisine is most likely to do that? To answer this we need to know something about the relative frequency of the different orderings. I'm going to test this out on number(salt,onions)/number(onions,salt) since that is the most popular pair of ingredients.
​
In what follows, I calculate a function N/M where N and M are integers. N is the number of times that 'salt,onions' happens for that cuisine and M is the number of times that 'onions,salt' happens. I calculate the ration N/M but I also calculate the error by propagating errors. In general I know that the percent error for a function $f(N,M)$ is just $\frac{\sqrt{\sigma_N^2 \frac{\partial f}{\partial N}^2 + \sigma_M^2 \frac{\partial f}{\partial M}^2}}{f(N,M)}$. For the function f=N/M the percent error is $\sqrt{\frac{\sigma_N^2}{N^2}+\frac{\sigma_M^2}{M^2}}$ so the total error is just $\frac{N}{M}\sqrt{\frac{\sigma_N^2}{N^2}+\frac{\sigma_M^2}{M^2}}$.

def construct_pairs_unordered(ingredients):
    ingredients2=[i for i in ingredients]
    output=[]
    for i,v1 in enumerate(ingredients):
        for j,v2 in enumerate(ingredients2):
            if i!=j:
                output.append(v1+str("----")+v2)
    return output
